# Проект 4: Визуализация сети контактов

**Сценарий**  
Создайте сервис для построения и отображения социального графа – кто с кем знаком в классе/группе. Пользователи (узлы) могут добавляться, между ними можно устанавливать связи (дружба). Сервис должен визуализировать граф (например, с помощью библиотеки D3.js или простого канваса) и позволять управлять данными через API.

**Роли и распределение задач**

| Ученик A (Бэкенд, графовая модель) | Ученик B (Фронтенд, визуализация) |
|------------------------------------|-----------------------------------|
| Создать модели: `Person` (`id`, `name`) и `Connection` (пара `person1_id`, `person2_id`) | Подготовить HTML-страницу с областью для рисования графа |
| Реализовать эндпоинты: добавление человека, добавление связи, получение списка людей и связей | Написать JavaScript-код для отрисовки узлов и рёбер на основе данных с сервера |
| Хранить данные в `network.json`: два списка – `"people"` и `"connections"` | Обеспечить форму для добавления новых людей и связей через интерфейс |
| Реализовать логику проверки уникальности и непротиворечивости связей | Использовать библиотеку vis.js или D3.js (можно готовый пример адаптировать) |

**Технические детали и подсказки**

- `network.json`:
```json
{
  "people": [
    {"id": 1, "name": "Анна"},
    {"id": 2, "name": "Борис"}
  ],
  "connections": [
    {"from": 1, "to": 2}
  ]
}
```

- Визуализация с vis.js (проще всего):
```html
<script type="text/javascript" src="https://unpkg.com/vis-network/standalone/umd/vis-network.min.js"></script>
<div id="mynetwork" style="height: 500px;"></div>
<script>
  const nodes = new vis.DataSet([{id:1, label:"Анна"},{id:2, label:"Борис"}]);
  const edges = new vis.DataSet([{from:1, to:2}]);
  const container = document.getElementById('mynetwork');
  const data = { nodes, edges };
  const options = {};
  const network = new vis.Network(container, data, options);
</script>
```

- API FastAPI:
  - GET /people – возвращает список людей.
  - POST /people – добавляет нового человека.
  - GET /connections – список связей.
  - POST /connections – добавляет связь.

**Тестирование**
- Добавьте 5–6 человек и несколько связей между ними.
- Проверьте, что при добавлении нового человека он появляется на графе.

**Критерии приёмки**
- Можно добавлять людей и связи через API.
- Визуализация графа обновляется автоматически или по кнопке.
- Данные сохраняются в JSON и не теряются при перезапуске.
